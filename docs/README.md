# Docs

Техническая документация, ADR и эксплуатационные заметки.

## Рекомендуемая структура

```text
docs/
├── ARCHITECTURE.md
├── adr/
├── runbooks/
└── diagrams/
```

## Принцип

- Архитектура - в одном master-документе.
- ADR фиксируют решения, которые могут измениться.
- Runbooks хранят операционные шаги без смешения с кодом.
